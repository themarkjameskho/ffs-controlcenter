# Control Center Appearance Lock

Status: **ACTIVE**

This project's current visual appearance is **final** and **frozen** until the owner explicitly says otherwise.

## Locked (Do Not Change)

- Color palette (all UI colors, including text, backgrounds, borders, accents)
- Container styling (card/container colors, fills, border colors, surface treatments)
- Visual theme/look-and-feel of the existing Control Center interface

No one should change these without explicit approval from the owner.

## Allowed (Can Change)

- Display details and information architecture, as long as appearance stays the same.
- Text hierarchy/content wording
- Which fields are shown in cards/modals
- Ordering/grouping of displayed data
- Interaction behavior and logic (drag/drop behavior, refresh/polling, data rendering)

Allowed changes must preserve the locked visual appearance above.

## Change Control Rule

If a task requires visual appearance changes, stop and request owner approval first.

Approval trigger phrase:

`Appearance changes approved by owner.`

Until that approval is given, treat appearance changes as out of scope.
